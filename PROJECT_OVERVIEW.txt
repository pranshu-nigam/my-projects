ZenPose - Simple Project Overview

This file explains what the project uses and how to run it. It has two parts: Frontend and Backend.

FRONTEND (What the user sees)
- What it is: The website/app that runs in your browser.
- What we used:
  - Node.js (version 16 or newer; 18+ is better)
  - Vite (fast dev server and build tool)
  - Tailwind CSS + PostCSS + Autoprefixer (styling and utilities)
  - Plain JavaScript (Vanilla JS with ES modules; we are not using React/Vue)
  - Browser APIs: getUserMedia (webcam), Canvas (drawing), Fetch (API calls)
- How it works (simple):
  - We ask for camera permission.
  - We read frames from the webcam and extract body keypoints.
  - We show the predicted pose and feedback on screen.
  - We send progress data to the backend using simple fetch() requests.
- Important files:
  - frontend/src/app.js (starts the app and wires pages)
  - frontend/src/components/ (UI parts)
    - AIPose.js, AIPoseImproved.js (live pose view + feedback)
    - Dashboard.js, Progress.js (history and stats view)
    - LoginPage.js, SignupPage.js, LeftPanelSignup.js (auth screens)
    - Header.js, Settings.js, Toast.js, ZenPoseLogo.js (UI helpers)
  - frontend/src/styles/
    - main.css (global styles), ai-pose-improved.css (pose view styles)
  - frontend/index.html + other demo/test HTML files (quick manual testing)
  - frontend/tailwind.config.js, frontend/postcss.config.js (styling setup)
- State and data:
  - We keep small bits of state in memory inside components (simple JS objects).
  - We can store lightweight items in localStorage if needed (e.g., tokens).
- How to run (development):
  1) Open a terminal and go to the folder: cd frontend
  2) Install packages: npm install
  3) Start the dev server: npm run dev
  4) Open the link shown (usually http://localhost:5173)
- How to build for production:
  - Run: npm run build
  - The built files will be in frontend/dist (use these for deployment)
- Notes:
  - Because we use the camera, production sites must be served over HTTPS for permissions to work reliably.

BACKEND (Server and API)
- What it is: The Python server that provides APIs (login, saving progress, etc.). It can also help with pose processing.
- What we used:
  - Python 3.10 or 3.11 (recommended)
  - FastAPI (web framework for building APIs)
  - Uvicorn (server that runs the FastAPI app)
  - SQLAlchemy + Pydantic (database + request/response validation)
  - SQLite (simple file database for development)
  - Auth tools: python-jose (JWT tokens), passlib (password hashing), python-multipart (form uploads)
  - AI/ML: TensorFlow + Keras (model), OpenCV (image/array ops), NumPy, Pillow
- How it works (simple):
  - Users sign up and log in; we store hashed passwords (not plain text).
  - We issue a JWT token after login; the frontend sends it with API calls.
  - We save progress (e.g., pose attempts) into the database.
  - We load the Keras model (yoga_pose_5_model_3.keras) for pose classification.
- Important files:
  - backend/main.py (starts the FastAPI app and defines routes)
  - backend/auth.py (login/auth logic: hashing, tokens)
  - backend/models.py (database tables)
  - backend/schemas.py (Pydantic models for requests/responses)
  - backend/database.py (database engine and session management)
  - backend/pose_detection.py (pose/keypoint processing helpers)
  - backend/requirements.txt (Python packages list)
  - backend/zenpose.db (SQLite database file used in development)
- Typical API endpoints (examples):
  - POST /auth/signup — create account
  - POST /auth/login — get JWT token
  - GET /progress — list user progress (needs token)
  - POST /progress — add a progress record (needs token)
- How to run (development):
  1) Create and activate a virtual environment:
     - Windows: python -m venv .venv && .venv\Scripts\activate
     - Mac/Linux: python -m venv .venv && source .venv/bin/activate
  2) Install packages: pip install -r backend/requirements.txt
  3) Start the server: uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
  4) Open the docs in the browser: http://localhost:8000/docs
- Database notes:
  - We use SQLite during development (a single file).
  - For production, switch to PostgreSQL or MySQL.
- Config:
  - Put secrets (JWT key, DB URL) in environment variables or a .env file.

HOW IT WORKS (end-to-end)
- The browser uses your webcam.
- The app finds body keypoints and uses a model (yoga_pose_5_model_3.keras) to recognize the pose.
- The frontend shows live feedback.
- The backend saves your progress and handles login.

DEFAULT LOCAL LINKS
- Frontend: http://localhost:5173
- Backend: http://localhost:8000 (API docs at /docs)

WHO IS IT FOR
- People practicing yoga at home.
- Instructors and studios doing remote coaching.
- Companies running wellness programs.

NOTES ON SCALING LATER
- Host frontend on a CDN (use the files in frontend/dist).
- Run multiple backend instances behind a load balancer.
- Move database to PostgreSQL and add caching (Redis) if needed.
- Consider running the model in the browser (WebGL/WebGPU) to lower server load.

PRIVACY
- Camera access is only with user permission.
- We do not store raw video; we store minimal user data (profile and progress).
